Sobre

O Node.js tem design semelhante e é influenciado por sistemas como Ruby Event Machine e Twisted, do Python . O Node.js leva o modelo de evento um pouco mais longe. Apresenta um loop de eventos como uma construção de tempo de execução, e não como uma biblioteca. Em outros sistemas, sempre há uma chamada de bloqueio para iniciar o loop de eventos. Normalmente, o comportamento é definido por meio de retornos de chamada no início de um script e, no final, um servidor é iniciado por meio de uma chamada de bloqueio como EventMachine::run(). No Node.js, não existe essa chamada de início do evento. O Node.js simplesmente entra no loop de eventos após executar o script de entrada. O Node.js sai do loop de eventos quando não há mais retornos de chamada para executar. Esse comportamento é como o JavaScript do navegador - o loop de eventos está oculto do usuário.

Mais sobre o NodeJS

O HTTP é um cidadão de primeira classe no Node.js, projetado com streaming e baixa latência em mente. Isso torna o Node.js bem adequado para a fundação de uma biblioteca ou estrutura da Web.

O Node.js sendo projetado sem threads não significa que você não pode tirar proveito de vários núcleos em seu ambiente. Os processos filhos podem ser gerados usando nossa child_process.fork()API e são projetados para facilitar a comunicação. Construído sobre a mesma interface está o clustermódulo, que permite compartilhar soquetes entre processos para permitir o balanceamento de carga em seus núcleos.
